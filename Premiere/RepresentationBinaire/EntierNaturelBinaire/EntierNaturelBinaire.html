Représentation binaire d'un entier naturel

Principe du binaire

Dans les domaines de l'automatisme, de l'électronique et de l'informatique, nous utilisons la base 2. Tous les nombres s'écrivent avec deux chiffres uniquement (0 et 1) car cela correspond à un système à deux états, par exemple :

  • Un interrupteur est ouvert ou fermé
  • Une diode est allumée ou éteinte
  • Une tension est présente ou absente
  • Une surface est réfléchissante ou non
  • Un champ magnétique est orienté Nord-Sud ou Sud-Nord

Le chiffre binaire qui peut prendre ces deux états est nommé "Bit" (Binary digit). On fait correspondre les nombres décimaux aux nombres binaire en les écrivant dans l'ordre croissant :

Pour ne pas confondre les deux représentations, on écrit les nombres dans des parenthèses en ajoutant la base en indice. Ainsi (101)10 est différent de (101)2 = (5)10.

Conversion décimal vers binaire

Pour écrire en base 2 un nombre en base 10, on réalise la décomposition du nombre grâce à des puissance de 2.

Exemple : 121 = 64 + 32 + 16 + 8 + 1 = 26 + 25 + 24 + 23 + 20. Il ne reste plus qu'à remplir le tableau :
2726252423222120
1286432168421
01111001
On a donc (121)10 = (0111 1001)2

  1. (100)10
  2. (150)10
  3. (666)10
  4. (737)10

Il existe une autre méthode qui ne nécessite pas de connaître les puissances de 2 : la méthode par divisions.
Cette méthode donne la représentation binaire du nombre en décimal bit par bit en partant de la droite en suivant les étapes suivantes :

  1. Si le nombre est impair, le bit suivant est un 1, sinon c'est un 0
  2. Répéter l'opération avec le quotient entier de la division du nombre par 2
Le processus s'arrête dès que le quotient entier est égal à 0.

Exemple pour (121)10 :

  • 121 est impair, le dernier bit sera donc 1. On considère maintenant 121 // 2 = 60.
  • 60 est pair, les derniers bits seront donc 01. On considère maintenant 60 // 2 = 30.
  • 30 est pair, les derniers bits seront donc 001. On considère maintenant 30 // 2 = 15.
  • 15 est impair, les derniers bits seront donc 1001. On considère maintenant 15 // 2 = 7.
  • 7 est impair, les derniers bits seront donc 1 1001. On considère maintenant 7 // 2 = 3.
  • 3 est impair, les derniers bits seront donc 11 1001. On considère maintenant 3 // 2 = 1.
  • 1 est impair, les derniers bits seront donc 111 1001. On considère maintenant 1 // 2 = 0. Le processus s'arrête.
Donc (121)10 = (0111 1001)2
