Représentation binaire d’un entier relatif.

On souhaite désormais coder des entiers positifs mais aussi des entiers négatifs.

Première approche

Une première approche simple consisterai à indiquer dans le premier bit le signe : 0 pour un nombre positif et 1 pour un nombre négatif par exemple.

Sur un octet, combien peut-on alors coder de nombres entiers relatifs ?
Avec cette méthode, représenter (si possible), les nombres :
  1. (17)10
  2. (-15)10
  3. (130)10
  4. (-1)10

Gros inconvénient de cette méthode : il existe deux zéros, l’un positif (0000 0000) et l’autre négatif (1000 0000).

Complément à 2 sur 1 octet

Le principe

Toujours sur 1 octet, voici une deuxième méthode plus pratique à implémenter dans un micro-processeur. Cette méthode utilise le dépassement de mémoire lors de la somme de deux entiers codés sur 1 octet comme nous l’avons vu précédemment.

Avec cette méthode, (121)10 se code (0111 1001)2 alors que (-121)10 se code (1000 0111)2.
Vérifier que, si l'on ignore le dépassement de mémoire, on obtient bien en binaire 121 + (-121) = 0.

Conversion décimal vers binaire

Cette méthode ne change pas la façon de coder les nombres positifs (penser quand même à le représenter avec 8 bits). Pour les nombres négatifs, il suffit de retirer, en binaire, la représentation de l’opposé du nombre à 1 0000 0000.

Avec cette méthode, représenter, les nombres :
  1. (17)10
  2. (-15)10
  3. (-66)10
  4. (-1)10

Mais on préférera retenir la petite routine :

  • Pour les nombres positif : le premier bit est 0, le reste est la représentation habituelle.
  • Pour les nombres négatifs :
    1. coder sur 1 octet l'opposé du nombre.
    2. réécrire en inversant la valeur de chaque bit.
    3. ajouter 1 à ce résultat.

Avec cette méthode, représenter, les nombres :
  1. (-17)10
  2. (111)10
  3. (-116)10
  4. (-5)10

Conversion binaire vers décimal

Il suffit de déconstruire la routine précédente :

  • Si la représentation binaire commence par 0, le nombre est positif. La conversion binaire vers décimale des nombres entiers naturels suffit.
  • Si la représentation binaire commence par 1, le nombre est négatif et :
    1. on réécrit le nombre en inversant la valeur de chaque bit
    2. on ajoute 1 à ce résultat
    3. on utilise la conversion habituelle des nombres entiers naturel
    4. le nombre obtenu est l'opposé du nombre cherché.

Avec cette méthode, convertir en décimal les nombres :
  1. (1000 0001)2
  2. (0011 1111)2
  3. (1011 1001)2
  4. (1111 0010)2

Complément à 2 sur n bits

La méthode précédente se généralise en codant les entiers sur n bits avec n > 2 (la plupart du temps, n est un multiple de 8).

Avec la méthode du complément à 2 sur 2 octets, représenter les nombres :
  1. (17)10
  2. (-500)10
  3. (1000)10
  4. (-2022)10
